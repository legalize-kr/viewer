GitHub 접근 토큰이 무엇인가요?

GitHub 접근 토큰은 GitHub가 사용자를 확인할 때 쓰는 개인용 접근 키입니다. 비밀번호가 아니며, 이 뷰어에서는 GitHub 요청 한도를 늘리고 공개 원문 목록과 변경 내역을 더 안정적으로 읽기 위해 사용합니다.

어떤 권한이 필요한가요?

문서를 읽기만 하므로 쓰기 권한은 필요하지 않습니다. Fine-grained personal access token을 만들고, Repository access는 공개 저장소 읽기 또는 필요한 저장소만 선택, Repository permissions의 Contents는 Read-only로 두세요.

  1. GitHub에 로그인한 뒤 프로필 메뉴에서 Settings를 엽니다.
  2. Developer settings → Personal access tokens → Fine-grained tokens로 이동합니다.
  3. Generate new token을 누르고 이름, 만료일, 저장소 접근 범위를 정합니다.
  4. Contents 권한을 Read-only로 설정한 뒤 Generate token을 누릅니다.
  5. 생성 직후 보이는 토큰을 복사해 설정의 GitHub 접근 토큰 칸에 붙여넣고 저장합니다.

토큰은 생성 직후 한 번만 볼 수 있습니다. 다른 사람에게 공유하지 말고, 필요 없어진 토큰은 GitHub 설정에서 삭제하세요.
